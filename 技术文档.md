# 中驰智能PPT生成平台技术文档

## 1. 当前业务口径

本系统面向售前 PPT 初稿生成。用户只需要创建项目、填写基础信息、统一上传项目资料，系统负责资料解析、项目类型识别、模板选择、案例匹配、字段替换、PPT 合并和下载。

正式流程不要求用户选择资料属于哪个章节。前端只负责项目管理、统一上传、识别结果确认、生成状态和下载。后端负责文件存储、解析、分类、模板选择、案例匹配、章节渲染和最终合并。

当前模块口径：

| 模块 | 当前处理方式 | 说明 |
|---|---|---|
| M1/M2 | 固化模板字段替换 | 共用项目类型模板；LLM 只辅助分类，不生成整章内容 |
| M3 | 正式流程支持模板生成 | 使用 M3 正式模板替换文字和已保存资料；图片相关另有功能测试入口 |
| M4 | 暂不参与当前生成链路 | 后续基于工程量、用钢量、工期等真实数据生成 |
| M5 | 真实案例匹配 | 来自案例库或历史资料，不允许编造案例 |
| M6 | 固定企业模板 | 默认使用企业介绍与荣誉背书模板 |

## 2. 主流程

1. `POST /api/projects` 创建项目。
2. `POST /api/projects/{project_id}/files` 统一上传项目资料。
3. `POST /api/projects/{project_id}/analyze` 解析资料、识别资料角色、判断项目类型、选择 M1/M2 模板、推荐 M5 案例。
4. 前端展示识别结果；M1/M2 卡片可点击“查看分析依据”，展示分类方式、LLM 理由、fallback 原因、关键词和证据片段。
5. `POST /api/projects/{project_id}/classification/review` 提交人工确认结果。
6. `POST /api/projects/{project_id}/generate` 生成 `M1/M2 -> M3 -> M5 -> M6`，未选择的 M3/M5 会跳过。
7. `GET /api/projects/{project_id}/download` 下载最终 PPTX。

## 3. 技术组件

| 组件 | 技术 | 职责 |
|---|---|---|
| 前端 | Next.js | 单页工作台、正式流程、功能测试入口 |
| 后端 | FastAPI + uv | API、文件记录、任务状态、分析与生成调度 |
| PPT 引擎 | python-pptx | 模板复制、字段替换、章节合并 |
| 存储 | 本地 JSON + 本地文件 | Demo 阶段使用，后续可替换 PostgreSQL 和对象存储 |
| 案例库 | 本地目录 + JSON 索引 | M5 案例推荐和来源追溯 |
| 向量库 | pgvector 预留 | 优先用于 M5 案例召回，后续扩展到素材和项目资料 |
| LLM | OpenAI 兼容接口 | M1/M2 分类、摘要、标签和缺失字段辅助判断 |

Python 项目默认使用 `uv` 管理环境和依赖。涉及深度学习环境时再使用本机 miniconda。

## 4. 模板策略

统一模板目录：

```text
D:\中驰股份\code\ppt_engine\templates\solution_fixed_modules
```

M1/M2 固定模板：

| project_type | 模板文件 |
|---|---|
| `highway` | `公路全封闭声屏障（M1_&_M2）.pptx` |
| `metro` | `轨道交通地铁全封闭声屏障（M1_&_M2）.pptx` |
| `existing_rail_transit` | `铁路_&_轨道交通既有线声屏障_（M1_&_M2）.pptx` |
| `railway` | `铁路声屏障行业背景与技术发展（M1_&_M2）.pptx` |

其他模板：

- M3 正式模板：`M3_项目深化方案模板.pptx`
- M3 完整测试模板：`M3_项目深化方案模板_最终版本.pptx`
- M5 模板：`M5示例.pptx`
- M6 模板：`中驰企业介绍合并初版（M6）.pptx`

模板原则：

- M1/M2 不由 LLM 从零生成。
- M5 必须来自真实案例或演示固定案例。
- M6 不由 LLM 编造资质、荣誉和业绩。
- 缺失字段统一保留为 `[待补充：字段名]`。
- M3 完整功能测试使用独立测试模板，不修改正式 M3 模板。

## 5. M1/M2 LLM 分类器

M1/M2 分类器使用“模板画像 + LLM 优先 + 规则 fallback”。

输入：

- 项目基础信息。
- 文件名和解析文本摘要。
- 4 个 M1/M2 模板画像。

输出：

- `detected_project_type`
- `confidence`
- `matched_keywords`
- `detection_evidence`
- `classification_method`
- `llm_reasoning_summary`
- `fallback_reason`

约束：

- 结果必须落在 `highway`、`metro`、`existing_rail_transit`、`railway`。
- LLM 配置缺失、请求失败、JSON 非法、枚举非法、置信度过低时自动使用规则 fallback。
- 环境变量 `ZHONGCHI_M1M2_LLM_ENABLED=0` 可关闭 M1/M2 LLM 分类。
- 第一版不接向量库；只有模板数量或模板页级匹配复杂度增加时，再考虑模板文本入库。

## 8. M3 边界

正式流程：

- 用户在“M3资料上传”维护 9 个部分的文字和图片资料。
- 生成时根据 `m3_selection` 决定是否包含 M3。
- 选择 `m3_template` 时渲染 M3 正式章节；选择 `m3_skip` 时跳过。

功能测试：

- `M3完整测试` 验证 9 部分文字 + 图片、多图扩页。
- 功能测试输出不写入正式项目，不影响正式生成链路。

## 9. 案例库和向量库

案例库目录：

```text
D:\中驰股份\code\data\case_library
```

案例推荐必须来自：

- `data/case_library` 的结构化案例。
- `D:\中驰股份\SR智能PPT拆分` 的历史资料索引。
- 用户确认的演示固定案例。

没有可用案例时必须返回无高匹配结果，不允许编造。

向量库现状：

- 已提供可选的向量入库/索引能力：`POST /api/projects/{project_id}/vector-index`。
- 命令行工具 `asset_tools build-vector` 可批量构建向量索引。
- 环境变量：`ZHONGCHI_VECTOR_DSN`（pgvector 连接串）、`ZHONGCHI_EMBEDDING_*`（Embedding 配置）。
- 未配置 DSN 或 Embedding 时自动走关键词 fallback，不阻塞主流程。
- 当前未正式投入生产检索，后续可扩展到素材、历史 PPT 和项目资料的统一检索。
