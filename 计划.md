# 中驰智能PPT规划与下一阶段深入方向

## 1. 目标

本文档用于承接当前 Demo 后续完善工作。后续改造继续以“统一上传项目资料，后端自动解析、分类、识别项目类型、选择模板、匹配案例、生成并合并 PPT”为主流程，不再回到“前端按 M1/M2/M5/M6 分模块上传”的旧方案。

本阶段规划重点是先补强资料解析、结构化分类、案例库与向量库，再在明确边界内接入大模型。大模型只能辅助摘要、标签提取、类型判断、缺失字段识别和案例匹配理由生成，不能编造案例、工程量、工期、资质荣誉，也不能直接生成 `M1/M2/M6` 整章内容。

## 2. 最新流程理解

1. 用户创建项目并填写基础信息。
2. 用户统一上传全部项目资料，不选择资料属于 M 几。
3. 后端保存原始文件，生成文件记录。
4. 后端解析 PDF、Word、Excel、PPT 等资料文本和元数据；图片、扫描件、CAD 先标记为待增强。
5. 后端输出每个文件的资料角色、可服务模块、置信度、命中依据和缺失字段线索。
6. 后端识别项目类型，并为 `M1/M2` 选择固化模板。
7. 后端从项目资料中提取标签，为 `M5` 匹配真实案例。
8. 后端为 `M6` 使用固定企业背书模板。
9. 前端展示识别结果，用户确认或修正项目类型、模板、案例和缺失字段。
10. 后端按确认结果执行字段替换、案例填充、模板复制和 PPT 合并。
11. 最终 PPTX 按 `M1/M2 -> M5 -> M6` 合并输出。
12. `M3/M4` 作为后续动态模块，当前阶段不进入生成链路。

## 4. 文档解析完善方向

多格式解析应作为下一阶段基础能力，不应直接跳到大模型生成。

- PDF：优先提取可复制文本；扫描件或图片型 PDF 标记为 `pending_ocr`；后续再接 OCR。
- Word：提取正文、标题、表格文字和段落结构。
- Excel：提取 sheet 名、表头、关键单元格、工程量表和参数表；不要只拼成一段纯文本。
- PPT：提取每页标题、正文、备注、图片占位信息，用于历史案例和素材索引。
- 图片：先记录文件名、尺寸、可能用途；后续接 OCR 和图像理解。
- CAD：先保存文件并标记为 `pending_cad_parse`；后续再接 CAD 解析或人工标注。

解析结果需要支持后续检索和追溯，建议形成文本 chunk，并保留 `source_file_id`、页码、sheet、slide、段落位置等来源信息。

## 5. 案例库与向量库路线

固定案例库目录：

```text
D:\中驰股份\code\data\case_library
```

案例库优先支持结构化案例，每个案例建议包含：

- `case_info.json`：案例 ID、标题、项目类型、城市、线路、施工场景、痛点、技术难点、标签、摘要、来源文件。
- 原始案例材料：PPT、PDF、Word、图片等。
- 解析文本和 chunk：用于关键词检索、Embedding 和证据追溯。

向量库分三步推进：

1. 阶段一：继续使用关键词/标签 fallback，保证 Demo 不依赖外部模型也能跑通。
2. 阶段二：对案例摘要和案例 chunk 建 embedding，优先服务 `M5` 案例召回。
3. 阶段三：把素材、历史 PPT、项目资料 chunk 纳入统一检索，用于资料复用、案例推荐和后续 M3/M4 动态模块。

## 6. 大模型接入边界

可以接入大模型的位置：

- 资料摘要：从解析文本中提炼摘要、关键字段和疑似缺失项。
- 资料角色判断：辅助判断 `project_brief`、`drawing`、`tender`、`survey`、`case_material`、`enterprise_material`、`unknown`。
- 标签提取：提取城市、线路、交通类型、施工场景、痛点、技术难点、工期条件。
- 项目类型判断：辅助判断 `highway`、`railway`、`metro`、`existing_rail_transit`，但最终必须落在固定枚举内。
- 案例匹配理由：基于真实案例库和命中证据生成可解释理由。
- 缺失字段识别：指出生成 PPT 前需要人工确认的字段。

禁止或不应强依赖大模型的位置：

- 不允许编造案例、业绩、资质、荣誉、工程量、用钢量、工期。
- 不允许让 LLM 从零生成 `M1/M2` 整章内容；`M1/M2` 只选固化模板并做字段替换。
- 不允许让 LLM 从零生成 `M6` 企业背书内容；`M6` 默认使用固定模板。
- 不允许 LLM 替代人工确认；识别结果必须可由用户确认或修正。
- LLM 不可用时，Demo 必须通过关键词、规则和 Mock fallback 跑通。

- 资料解析结果能支撑项目类型识别、模板选择、案例匹配和缺失字段提示。
