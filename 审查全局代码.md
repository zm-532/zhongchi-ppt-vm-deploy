## 五、按优先级的修复建议

**立即处理（安全）：**

1. 轮换所有已暴露的 API 密钥
2. 添加 API Key 中间件认证
3. 添加文件上传大小限制
4. 清理 `asset_tools` 中重复且不安全的 PgVectorStore

**短期改进（稳定性）：** 5. 用 SQLite 替换 JSON 文件存储，或至少加文件锁 6. 收窄异常捕获范围，区分业务异常和编程错误 7. 错误信息只返回通用描述，详细信息仅记录服务端日志 8. 前端添加 `AbortController` 和请求竞态保护

**中期重构（可维护性）：** 9. 拆分前端单体组件（按视图/功能模块） 10. 将相关 state 整合为 `useReducer` 或自定义 Hook 11. 添加 `useCallback`/`React.memo` 优化渲染性能 12. 补充业务逻辑单元测试和错误场景 E2E 测试 13. 清理已提交的 `.log` 和 `.tsbuildinfo` 文件
