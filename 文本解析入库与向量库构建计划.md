# 文本解析入库与向量库构建计划

## 概要
目标是在现有 `code/backend` 的文件解析流程后，把解析文本切块、生成 embedding，并写入 `pgvector` 向量库；同时补齐 `code/asset_tools` 中当前占位的 `PgVectorStore`，让素材匹配从“关键词 fallback”升级为“向量优先、关键词兜底”。

当前仓库现状：
- 解析入口：`code/backend/src/app/document_analysis.py`
- 上传/分析主流程：`code/backend/src/app/storage.py`
- 向量库占位：`code/asset_tools/asset_tools/vector_store.py`
- 测试：`code/backend/tests/test_api.py`、`code/asset_tools/tests/test_assets.py`

## 接口与数据约定
- 环境变量：
  - `ZHONGCHI_VECTOR_DSN`：PostgreSQL DSN，未配置则不启用向量库。
  - `ZHONGCHI_VECTOR_TABLE`：默认 `zhongchi_embeddings`。
  - `ZHONGCHI_EMBEDDING_DIM`：测试默认 `1536`。
  - `ZHONGCHI_EMBEDDING_PROVIDER`：默认 `hash`，生产可扩展为 `openai_compatible`。
- 向量表字段：
  - `id bigserial primary key`
  - `doc_type text`：`project_file` 或 `asset`
  - `project_id int null`
  - `file_id int null`
  - `asset_id int null`
  - `module_id text`
  - `chunk_index int`
  - `title text`
  - `source_path text`
  - `text text`
  - `metadata jsonb`
  - `content_hash text unique`
  - `embedding vector(1536)`
  - `created_at timestamptz`
  - `updated_at timestamptz`
- 幂等策略：同一 `content_hash` 再次写入时更新 `text`、`metadata`、`embedding`、`updated_at`。
- 失败策略：向量库不可用或写入失败时，不影响现有解析、分类、PPT 生成流程；记录 `embedding_status` 或 error message，继续关键词匹配。
